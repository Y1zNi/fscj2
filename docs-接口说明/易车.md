# 易车（接口字段说明）

---

## 1. 接口一：个人主页信息流 JSON（Flow）

**`POST https://mapi.yiche.com/web_api/flow_api/api/v1/flow/user_home_page`**

**Content-Type**：`application/json;charset=UTF-8`

### 1.1 请求头（常见）

| 头 | 说明 |
|----|------|
| `cid` | 与 Body 中 `cid` 一致；页面 `window.baseConf.cid` 常见为 `"601"`（渠道/客户端标识） |
| `origin` | 常见 `https://i.m.yiche.com` |
| `referer` | 常见 `https://i.m.yiche.com/u{userId}/newcenter/!all/`（与 M 站「全部」Tab 一致） |
| `user-agent` | 移动端 Safari 等 |
| `x-timestamp` | 请求时刻毫秒时间戳（字符串） |

### 1.2 请求体

```json
{
  "cid": "601",
  "param": {
    "userId": "27081208",
    "timestamp": "1776244298692"
  }
}
```

| 字段 | 说明 |
|------|------|
| `cid` | 渠道 id，与页面 `baseConf`、请求头 `cid` 对齐 |
| `param.userId` | 作者数字 id（字符串即可） |
| `param.timestamp` | 翻页游标：**首屏**常见为当前毫秒时间戳；**下一页**用上一页响应里 `data.timestamp` 或 `data.lastRefreshTime` |

### 1.3 响应顶层

| 字段 | 说明 |
|------|------|
| `status` | 成功常见为字符串 `"1"` |
| `message` | 如 `success` |
| `data` | 业务数据 |
| `ercd` | 错误码扩展，成功常为空字符串 |

### 1.4 `data` 层

| 字段 | 说明 |
|------|------|
| `timestamp` | 供下一页请求的游标（毫秒，字符串或数字均可再转字符串） |
| `lastRefreshTime` | 与 `timestamp` 类似，可作翻页兜底 |
| `vestUserFlag` | 马甲/小号类标记（布尔） |
| `list` | 信息流卡片数组（文章、视频、点评等混排） |

### 1.5 `data.list[]`（单条：字段随 `type` 变化）

以下为接口中**常见可取值**（并非每条都有全部字段）。

| 字段 | 说明 |
|------|------|
| `id` | 内容 id（字符串） |
| `title` | 标题 |
| `type` | 内容类型码；文章常见 `21`，视频常见 `4`（以实际返回为准） |
| `userId` | 作者 id |
| `user` | 作者信息（昵称、头像、粉丝、简介、`roles` 等） |
| `publishTime` | 发布时间展示串（如 `2026-04-13 15:33:14` 或仅日期） |
| `publishTimestamp` | 发布时间毫秒时间戳 |
| `ts` | 紧凑时间串（如 `20260413153314`） |
| `summary` | 短摘要 |
| `contentSummary` | 要点式摘要（多行编号） |
| `newsDetail` | **文章正文结构**：JSON **字符串**，解析后为块数组（见第 2 节） |
| `mLinkUrl` | M 站主链接（文章常为 `https://hao.m.yiche.com/wenzhang/{id}/`） |
| `newsUrl` | 相对路径形式（如 `/hao/wenzhang/{id}/`） |
| `pcLinkUrl` | PC 域文章链 |
| `commentCount` / `supportCount` / `supportStatus` | 评论、点赞等 |
| `coverList` / `verticalCoverList` / `detailCoverList` / `coverImgs` | 封面与图集 |
| `serialId` / `serialName` / `serialIds` / `serialInfoList` | 关联车系 |
| `shareData` | 分享用（标题、摘要、link、小程序路径等） |
| `newsDetail` 外视频类常见 | `mp4Link`、`duration`、`resource[]`、`playCount`、`visitCount`、`url`（视频页）等 |

### 1.6 与 M 站 HTML 的关系

浏览器打开 **`GET https://i.m.yiche.com/u{userId}/newcenter/!all/`** 会先拿到 **HTML 壳子**，再由前端请求本接口拉取 `list`。  
单独调用本接口时，只要 **`userId` + `cid` + 合法 `timestamp`** 即可，不必先请求 HTML；`referer` 建议与 M 站主页一致。

## 2. `newsDetail`：文章正文块结构

`newsDetail` 为 **字符串**，需 **`JSON.parse`** 得到 **数组**。每一项为一块内容，用 **`type`** 区分块类型。

### 2.1 常见块类型（能力面）

| `type` | 含义（常见） |
|--------|----------------|
| `1` | 正文段落，`content` 为文字（可带 `style` 做加粗、内链等） |
| `2` | 图片块：`content` 或空、`contentWp` 等为图地址 |
| `7` | 文内链接/样式（常与 `type=1` 的 `style` 组合出现） |
| `15` | 文末固定声明（如「内容由作者提供，不代表易车立场」） |
| 其它 | 以实际 JSON 为准 |

**拼纯文本正文**时，通常只抽取 **`type === 1`** 的 `content`；图片、声明等是否纳入取决于产品需求。

---

## 3. 抓取注意事项（通用）

| 项 | 建议 |
|----|------|
| Host | 接口域名为 `mapi.yiche.com` |
| `cid` | 与页面 `baseConf.cid` 一致；常见 `601` |
| `Referer` | 使用 `https://i.m.yiche.com/u{userId}/newcenter/!all/` 较贴近真实访问 |
| Cookie | 多数场景可不带；若返回空或风控再补 Cookie |
| 翻页 | 使用响应 `data.timestamp`（或 `lastRefreshTime`）作为下一页 `param.timestamp` |
| `newsDetail` | 为 JSON 字符串，需 parse；正文含多块时需按 `type` 过滤或拼接 |
| 混排 | `list` 中含视频（如 `type=4`）、文章（如 `type=21`）等，需按业务过滤类型 |
