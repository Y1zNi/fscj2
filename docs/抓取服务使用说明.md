# 飞书插件 · 本地抓取服务使用说明

本地抓取服务是一个自带前端静态资源与 API 的本地 HTTP 服务，供飞书多维表格插件通过页面地址访问（默认 `http://127.0.0.1:3789/`）。**无需单独安装 Node.js**（使用已打包的可执行文件时）。

**Windows 用户：** 见 **[Windows端抓取服务使用说明.txt](./Windows端抓取服务使用说明.txt)**。  
**Mac 用户（含 FSCJ-mac-arm64、终端步骤）：** 见 **[Mac端抓取服务使用说明.txt](./Mac端抓取服务使用说明.txt)**。

## 产物说明

| 平台 | 文件名 | 获取方式 |
|------|--------|----------|
| Windows | `FSCJ-抓取服务.exe` | 在本机执行 `npm run build:exe`，或从 GitHub Actions 工件「fscj-windows」下载 |
| macOS Apple芯片 (M 系列) | `FSCJ-mac-arm64` | 在 **macOS** 上执行 `npm run build:mac`，或从 GitHub Actions 工件「fscj-macos」下载 |
| macOS Intel | `FSCJ-mac-x64` | 同上 |

> **说明：** 在 Windows 上使用 `pkg` 交叉编译 macOS 目标常会失败（例如 `spawn UNKNOWN`）。**推荐在 macOS 上打 Mac 包，或使用仓库自带的 GitHub Actions**（推送 `main`/`master` 或在 Actions 里手动运行「Build desktop binaries」），从运行结果中下载 `fscj-macos` / `fscj-windows` 工件。

## 开发者：如何打包

前置：已安装 Node.js 18+、在项目根目录执行过 `npm ci`。

1. **仅 Windows 安装包（在 Windows 上）**
   ```bash
   npm run build:exe
   ```
   输出：`release/FSCJ-抓取服务.exe`

2. **仅 macOS（必须在 macOS 上，或交给 CI）**
   ```bash
   npm run build:mac
   ```
   输出：`release/FSCJ-mac-arm64`、`release/FSCJ-mac-x64`

3. **GitHub Actions**  
   将代码推送到 GitHub 后，打开 **Actions → Build desktop binaries**，等待完成后在 **Artifacts** 中下载对应平台的压缩包。

## 用户使用步骤

### Windows

1. 将 `FSCJ-抓取服务.exe` 放到任意文件夹（可选：把同版本构建的 `dist` 放在 exe **同目录**，可覆盖内置前端；一般只用 exe 即可）。
2. 双击运行。若防火墙提示，请允许专用网络访问。
3. 浏览器会自动打开 **控制台**；若未打开，请手动访问控制台地址（见下文「控制台与端口」）。
4. 在飞书多维表格插件中，将页面地址配置为控制台里显示的 **`http://127.0.0.1:端口/`**（注意末尾 `/` 与端口以实际为准）。

### macOS

1. 从工件或发布页取得 **`FSCJ-mac-arm64`**（M 芯片）或 **`FSCJ-mac-x64`**（Intel），放入任意文件夹。
2. 首次运行前在终端赋予执行权限（路径请改成你的实际位置）：
   ```bash
   chmod +x ./FSCJ-mac-arm64
   ```
   Intel 机型将文件名换成 `FSCJ-mac-x64`。
3. 运行：
   ```bash
   ./FSCJ-mac-arm64
   ```
4. 若提示「无法打开，因为来自身份不明的开发者」：
   - 打开 **系统设置 → 隐私与安全性**，在底部选择 **仍要打开**；或  
   - 右键点击文件 → **打开**，在弹出框中确认打开。  
   （未做 Apple 公证时属正常现象。）
5. 控制台一般会自动用默认浏览器打开；同样在飞书里把插件地址配成 **`http://127.0.0.1:端口/`**。

### 可选：外置 `dist` 目录

若可执行文件与 **`dist`** 文件夹（内含 `index.html`）放在**同一目录**，程序会优先使用外部的 `dist`，便于你单独更新前端而无需重新打包。

## 控制台与端口

- **控制台地址：** `http://127.0.0.1:<端口>/__control`  
- **插件页面地址：** `http://127.0.0.1:<端口>/`  
- **默认端口：** `3789`；若被占用，会自动尝试 `3790`～`3808`。  
- **修改端口：** 启动前设置环境变量 `PORT`（例如 `3789`）。  
- **不自动打开浏览器：** 设置环境变量 `NO_OPEN=1`。  
- **停止服务：** 在控制台页面点击「停止服务」，或关闭终端/结束进程。

## 常见问题

1. **插件连不上本地地址**  
   确认抓取服务已启动、端口未被占用、飞书里填的地址与控制台一致（含 `http://` 与端口）。

2. **Mac 上权限或安全拦截**  
   按上文「macOS」一节处理 Gatekeeper；仍失败时可在「隐私与安全性」中查看是否被拦截。

3. **Windows 上打不出 Mac 包**  
   请使用 macOS 本机执行 `npm run build:mac`，或使用 GitHub Actions 的 **fscj-macos** 工件。
